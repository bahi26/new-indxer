for root, use <code>su</code> and <code>busybox</code>, installed by default in Ubuntu. All of <code>su</code>'s required libraries are in <code>/lib</code>. Busybox is a collection of utilities that's statically linked, so missing libraries shouldn't be a problem. Do:</p><pre><code>su -c '/bin/busybox mv /usr_bak /usr'</code></pre><p>(While Busybox itself also has a <code>su</code> applet, the <code>/bin/busybox</code> binary is not setuid and so doesn't work unless ran as root.)</p><p>If you don't have a root password, you could probably <a href="https://unix.stackexchange.com/a/346642/70524">use Gilles' solution here using <code>LD_LIBRARY_PATH</code></a>, or reboot and edit the GRUB menu to boot with <code>init=/bin/busybox</code> and move the folder back.</p>    </div>    <div class="grid mb0 fw-wrap ai-start jc-end
